# CLI リファレンス

`knoll <subcommand> [...flags]`．

## `knoll run`

1 構成（rule または LLM モード）を `--runs` 回実行する．

| Flag | 既定 | 説明 |
|------|------|------|
| `--decision-mode <rule\|llm>` | `rule` | 排他：`rule` は `VoiceDecisionRule` を，`llm` は `VoiceDecisionLlm`（socsim-llm 駆動）を配線する． |
| `--n-teams <usize>` | `8` | チーム数． |
| `--team-size <usize>` | `12` | チームあたり従業員数（合計 = teams × size）． |
| `--n-levels <u8>` | `3` | 階層レベル数（記述専用）． |
| `--network <ws\|er\|ba>` | `watts-strogatz` | ネットワーク族． |
| `--network-k <usize>` | `6` | Watts–Strogatz の `k` / Barabási–Albert の `m`． |
| `--network-beta <f64>` | `0.1` | Watts–Strogatz の β / Erdős–Rényi の `p`． |
| `--motive-prior-as / -qs / -ps / -os <f64>` | `0.22 / 0.27 / 0.40 / 0.18` | 4 動機初期周辺分布（Knoll Study 2 平均）． |
| `--beta-psafety <f64>` | `1.2` | β_ψ — 心理的安全性の VOICE 係数． |
| `--beta-fear <f64>` | `1.5` | β_f — 恐怖の VOICE 係数（負号で使用）． |
| `--beta-rho <f64>` | `1.0` | β_ρ — 知覚近傍サイレンスの VOICE 係数． |
| `--beta-rho-ps <f64>` | `0.1` | **β_ρ^{PS}** — 動機 softmax 内 PS 行の ρ 係数．最重要ノブ． |
| `--prosocial-climate-decoupling` | `false` | β_ρ^{PS} = 0 に強制し，PS persona prompt から風土キューを除去 — Knoll の PS-風土独立性知見を機構的に再現． |
| `--p-retaliate <f64>` | `0.05` | agent あたり 1 ステップの報復確率． |
| `--shock-t <u64>` | （なし） | 任意の σ ショックステップ． |
| `--shock-magnitude <f64>` | `0.3` | σ ショックの大きさ． |
| `--t-max <u64>` | `36` | 最大シミュレーションステップ． |
| `--runs <usize>` | `1` | 独立実行回数（異なる seed；出力は *最終* 実行）． |
| `--seed <u64>` | `42` | ルート seed（決定論的 socsim コアを支配）． |
| `--llm-temperature <f32>` | `0.0` | LLM 生成温度． |
| `--llm-seed <u64>` | `0` | LLM seed オフセット；per-(agent, t) seed は派生． |
| `--llm-cache-path <path>` | `.llm_cache/cache.json` | プロンプト→応答キャッシュ（LLM モードのみ）． |
| `--output-dir <path>` | `results` | 出力ベースディレクトリ． |

出力（`results/<timestamp>/` 配下）:

- `config.json` — 実行構成．
- `metrics.csv` — 1 ステップ 1 行: `t, silence_rate, motive_mix_{AS,QS,PS,OS}, subscale_proxy_*, climate_of_silence, issue_salience, kl_divergence_to_knoll`．
- `agents.csv` — 最終ステップ agent 別状態．
- `correlations.csv` — 4 動機 × 6 correlate の Pearson r 行列（最終ステップ）．
- `run_metadata.json` — LLM 由来情報（model / endpoint / temp / seed / cache-hit 率）．

## `knoll sweep`

`β_ψ × β_f × β_ρ^{PS} × prosocial_decoupling × seeds` のデカルト積．`sweep_summary.csv` にセルごとの行を出力．

| Flag | 既定 | 説明 |
|------|------|------|
| `--decision-mode <rule\|llm>` | `rule` | β スイープが意味を持つのは `rule`． |
| `--beta-psafety-values <csv>` | `0.6,1.2,2.0` | β_ψ 値（カンマ区切り）． |
| `--beta-fear-values <csv>` | `0.5,1.5,2.5` | β_f 値． |
| `--beta-rho-ps-values <csv>` | `0.0,0.1,0.3` | β_ρ^{PS} 値． |
| `--sweep-decoupling <bool>` | `true` | `prosocial_climate_decoupling ∈ {false, true}` も併せてスイープ． |
| `--runs <usize>` | `5` | セルあたり実行回数． |
| `--t-max <u64>` | `36` | 1 実行あたり最大ステップ． |
| `--seed <u64>` | `42` | ベース seed；セル別 seed は派生． |
| `--output-dir <path>` | `results` | 出力ベースディレクトリ． |

出力（`results/<timestamp>_sweep/` 配下）:

- `sweep_config.json` — sweep 構成．
- `sweep_summary.csv` — `(セル × seed)` 1 行ずつ: `decision_mode, beta_*, prosocial_climate_decoupling, run, seed, final_round, silence_rate, motive_mix_*, climate_of_silence, corr_{AS,QS,PS,OS}_climate, kl_divergence_to_knoll`．

## `knoll reproduce`

Phase B3 / Phase X 用 stub（Phase B3 の予定内容を表示．Python 側の stub は `cfa-analysis` と `reproduce-paper` を参照）．

## `knoll-tools`（Python）

`knoll-tools <subcommand> [...flags]`．サブコマンド:

- **Track B（ABM）**: `visualize`, `visualize-sweep`, `show-experiment-settings`, `cfa-analysis`（stub）, `reproduce-paper`（stub）
- **Track A（心理測定）**: `survey-loader`, `descriptive-stats`, `efa-4factor`, `cfa-competing-models`, `reliability-analysis`, `nomological-network`, `discriminant-validity`, `robustness-checks`, `multigroup-cfa`

サブコマンドごとのフラグは `knoll-tools <subcommand> --help` で確認．
